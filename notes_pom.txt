2. Comment trouver et corriger une vulnérabilité
✅ Étape 1 — Scanner
mvn dependency-check:check

👉 Résultat :

fichier HTML dans :
target/dependency-check-report.html
✅ Étape 2 — Lire le rapport

Tu verras :

CVE ID (ex: CVE-2023-12345)
librairie vulnérable
version actuelle
version corrigée
✅ Étape 3 — Trouver version corrigée
🔎 Méthode 1 (rapide)

Google :

jackson-databind CVE fix version
🔎 Méthode 2 (pro)

Utilise :

Maven Central
GitHub release
✅ Étape 4 — Corriger
✔️ Cas 1 (Spring Boot gère déjà)

👉 Upgrade Boot

<version>4.1.0-M5</version>
✔️ Cas 2 (override manuel)
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.17.2</version>
</dependency>
⚠️ Règle PRO

👉 Toujours préférer :

Upgrade Spring Boot
Override seulement si nécessaire
🧠 Exemple concret
Vulnérabilité :
jackson-databind 2.15.x vulnérable
Fix :
2.17.x
Correction :
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.17.2</version>
</dependency>
🚀 Workflow pro complet
mvn clean install
mvn dependency-check:check
Lire rapport
Fix version
Rebuild
